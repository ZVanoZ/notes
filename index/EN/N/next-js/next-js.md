# "Next.js"

**Next.js** — это современный open-source фреймворк для веб-разработки, построенный поверх библиотеки **React**. Если React — это набор инструментов для создания интерфейса, то Next.js — это полноценный «завод», который берет на себя все сложные архитектурные задачи.

Он создан компанией **Vercel** и на сегодняшний день является стандартом индустрии для создания быстрых и SEO-оптимизированных сайтов.

---

### Ключевые особенности (почему его выбирают)

Главная «фишка» Next.js — гибкость в том, как именно ваш сайт доставляется пользователю. В обычном React сайт часто долго грузится (Client-Side Rendering), а в Next.js вы сами выбираете режим:

1. **Server-Side Rendering (SSR):** Страница генерируется на сервере в момент запроса. Это идеально для актуальных новостей или личных кабинетов.
2. **Static Site Generation (SSG):** Все страницы сайта создаются заранее при сборке проекта. Сайт работает молниеносно, как статический HTML. Подходит для блогов и документации.
3. **Incremental Static Regeneration (ISR):** Уникальная технология, позволяющая обновлять статические страницы в фоновом режиме без перезагрузки всего сайта.

---

### Основные преимущества

* **SEO-дружелюбность:** Поскольку сервер отдает уже готовый HTML, поисковики (Google, Яндекс) мгновенно индексируют контент.
* **Файловый роутинг:** Вам не нужно настраивать навигацию вручную. Просто создайте файл `about.tsx` в папке `pages` (или `app`), и адрес `vash-сайт.com/about` заработает автоматически.
* **Оптимизация изображений:** Компонент `<Image />` сам меняет размер картинки под экран пользователя и конвертирует её в современные форматы (WebP).
* **API Routes:** В Next.js можно писать серверный код (бэкенд) прямо внутри проекта. Вам не нужен отдельный сервер на Node.js или Python для простых задач вроде отправки формы.

---

### Сравнение: React vs Next.js

| Характеристика | React (CRA) | Next.js |
| --- | --- | --- |
| **Рендеринг** | Только в браузере (CSR) | SSR, SSG, ISR, CSR |
| **SEO** | Плохое (из коробки) | Отличное |
| **Маршрутизация** | Нужно ставить библиотеку | Встроена (по файлам) |
| **Бэкенд** | Нет | Есть (API Routes) |

---

### Архитектура App Router (Современный стандарт)

С выходом 13-й версии Next.js внедрил **App Router**, который основан на **React Server Components**. Это позволяет:

* Уменьшить объем JavaScript, отправляемого пользователю.
* Легко создавать вложенные макеты (layouts).
* Управлять загрузкой данных прямо внутри компонентов.

### Кому стоит учить Next.js?

Если вы уже знаете основы React, Next.js — это ваш следующий логичный шаг. Он востребован в бизнесе, так как напрямую влияет на скорость загрузки сайта и его позиции в поиске, что критично для интернет-магазинов и медиа-ресурсов.
