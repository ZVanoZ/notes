* Вопрос 

---


Как использовать Context7 в связке с "vanzan01/cursor-memory-bank"?

---

Использование **Context7 MCP** вместе с **Memory Bank** превращает ваш проект из «просто кода» в высокотехнологичную базу знаний. В этой связке Context7 выступает как **внешний эксперт**, а Memory Bank — как **хранилище принятых решений**.

Вот как интегрировать их в ваш рабочий процесс по шагам:

* Ответ 

---

### 1. Этап `/van` или `/plan`: Исследование

Когда вы только начинаете задачу (особенно уровня **Level 3**), вам нужно знать «как правильно сделать это сегодня», а не «как это делали в 2023 году».

* **Действие:** Попросите Агента использовать Context7 для изучения актуальных библиотек.
* **Промпт:** > `/plan Используй context7, чтобы найти последние рекомендации по использованию `dns-packet`в Node.js 22. На основе этих данных обнови`techContext.md`и составь план реализации в`tasks.md`.`
* **Результат:** Ваш `techContext.md` пополнится ссылками на актуальную документацию, и вы избежите устаревших паттернов.

### 2. Этап `/creative`: Выбор архитектуры

Context7 помогает сравнивать библиотеки в реальном времени.

* **Действие:** Сравните подходы, используя свежую документацию.
* **Промпт:**
> `/creative Используй context7, чтобы сравнить реализацию TLS-прокси в `node-forge`и встроенном модуле`tls`. Что из этого лучше подходит для нашего DNS-over-TLS проекта? Запиши обоснование в `systemPatterns.md`.`



---

### 3. Этап `/build`: Написание кода без галлюцинаций

Это самый важный момент. Вместо того чтобы выдумывать параметры функций, Агент будет «подсматривать» в официальные доки через MCP.

* **Действие:** При выполнении сложного шага принудительно направляйте Агента к докам.
* **Промпт:**
> `/build Реализуй Phase 3. Перед написанием кода используй context7, чтобы проверить сигнатуры методов в последней версии библиотеки `axios` (или любой другой из вашего стека).`



### 4. Этап `/reflect`: Проверка на соответствие (Best Practices)

Вы можете использовать Context7 как «судью», чтобы проверить, не написал ли Агент плохой код.

* **Промпт:**
> `/reflect Используй context7, чтобы найти официальный Style Guide для React-компонентов с TypeScript. Проверь наш новый `ErrorBoundary` на соответствие этим стандартам.`



---

### Где хранить данные Context7 в Memory Bank?

Чтобы не обращаться к MCP-серверу каждый раз (экономия времени и лимитов), сохраняйте важные выжимки в файлы памяти:

1. **`techContext.md`**: Сюда записывайте версии библиотек и прямые ссылки/сниппеты, которые нашел Context7.
2. **`style-guide.md`**: Если Context7 нашел новый стандарт оформления кода (например, для новой версии ESLint), зафиксируйте его здесь вручную или через Агента.
3. **`systemPatterns.md`**: Если Context7 подсказал крутой паттерн проектирования, опишите его здесь, чтобы Агент использовал его во всем проекте.

---

### Резюме связки:

* **Context7** — дает **внешнюю истину** (как надо делать сейчас).
* **Memory Bank** — фиксирует **внутреннюю истину** (как мы делаем это в нашем проекте).
