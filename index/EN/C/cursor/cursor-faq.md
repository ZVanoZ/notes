# Вопросы и ответы по программе Cursor 


---

* Вопрос

```text
Как правильно переключаться между ветками GIT, чтобы не ломать Codebase Indexing? 
```

* Ответ

Правильное переключение между ветками в Cursor — это залог того, что ИИ не будет «галлюцинировать», предлагая код из соседней ветки. Поскольку индекс завязан на физические файлы на диске, любая массовая смена файлов (что и делает `git checkout`) заставляет индексатор потеть.

Вот «золотой стандарт» того, как это делать правильно, чтобы сохранить рассудок вашего ИИ-агента:

### 1. Состояние "Clean Slate" (Чистый лист)

Перед переключением ветки **всегда** нажимайте **«Keep All»** или **«Discard All»** в чате.

* **Почему:** Если у вас висят незавершенные изменения в чате Cursor (режим diff), и вы переключите ветку через терминал, файлы на диске изменятся, а виртуальный diff в редакторе останется висеть поверх нового кода. Это приведет к критическому конфликту в «сознании» ИИ.

### 2. Синхронизация Memory Bank

Если вы используете `vanzan01/cursor-memory-bank`, перед уходом с ветки сделайте «фиксацию состояния»:

1. Введите команду `/archive` (если этап завершен) или просто убедитесь, что `activeContext.md` отражает текущий статус.
2. Сделайте `git commit`.

* **Важно:** Memory Bank — это тоже файлы. Если вы не закоммитите их на ветке А и перейдете на ветку Б, ваши файлы памяти могут «переехать» с вами (если нет конфликтов), смешивая контекст разных задач.

---

### 3. Правильный алгоритм переключения

Чтобы индекс обновился максимально корректно, следуйте этой схеме:

1. **Завершите сессию чата:** Нажмите кнопку «New Chat» (`Ctrl + L`) или просто убедитесь, что в текущем чате нет активных diff-ов.
2. **Переключитесь:** Выполните `git checkout <branch_name>`.
3. **Дождитесь индексации:** Посмотрите в правый нижний угол окна Cursor. Там может появиться крутящийся значок или надпись **"Indexing..."**.
* *Не задавайте вопросы Агенту, пока индекс не дойдет до 100%.* Иначе он может выдать ответ, основываясь на данных из предыдущей ветки.


4. **Сброс контекста:** После переключения на новую ветку в первом же запросе напишите Агенту:
> `@Codebase Я переключился на ветку [название]. Просканируй `memory-bank`, чтобы актуализировать текущий контекст задачи.`



---

### 4. Что делать, если индекс «сломался»?

Иногда после прыжков по веткам ИИ начинает утверждать, что в файле есть функция, которую вы удалили 10 минут назад. Это значит, что локальная база данных (SQLite) не успела обновиться.

**Как «пролечить» индекс:**

1. Зайдите в **Settings** (`Ctrl + Shift + J`).
2. Перейдите в раздел **General** -> **Features** -> **Codebase Indexing**.
3. Нажмите кнопку **"Rescan"** или **"Delete and Re-index"**.
* *Это удалит файлы индекса с диска и создаст их заново на основе текущего состояния файлов.*



### 5. Использование .cursorignore

Если у вас есть ветки с огромным количеством логов, временных файлов или `node_modules`, которые не нужны для работы ИИ, обязательно добавьте их в файл `.cursorignore`. Это ускорит переиндексацию при смене веток, так как Cursor будет игнорировать мусор.

---

### Резюме:

* **Никаких активных diff-ов** (нажать Keep All).
* **Коммит** в Git.
* **Пауза** в 5-10 секунд после перехода для работы индексатора.
* **Rescan**, если ИИ начал «врать».

* Вопрос

```text
Можно ли в Cursor вместо панели с чатами AI сделать отдельное окно?
Я хочу переместить окно на другой монитор и освобобить место для редактора кода.
```

* Ответ 

```
В Cursor можно вынести панель чата в отдельное окно. Варианты:

1. Через контекстное меню:
   - Правый клик по заголовку панели чата → "Move to New Window" или "Move Panel to New Window"

2. Через команды:
   - `Ctrl+Shift+P` (или `Cmd+Shift+P` на Mac)
   - Введите "View: Move Panel to New Window" или "Move Chat to New Window"
   - Выберите команду

3. Через настройки:
   - Откройте Settings (`Ctrl+,`)
   - Найдите настройки панели/чата
   - Ищите опции типа "Panel Location" или "Chat Panel Position"

4. Перетаскивание:
   - Попробуйте перетащить панель чата за заголовок за пределы окна Cursor — это может создать отдельное окно

После вынесения в отдельное окно его можно переместить на другой монитор и расположить рядом с основным окном редактора.

Если эти варианты не сработают, проверьте документацию Cursor или настройки 
интерфейса — функциональность может отличаться в зависимости от версии.
```

* Вопрос

```text
Почему в Cursor 2.3 нет пункта "Add custom Mode" при выборе режима в чате AI?
Доступно только: Agent; Plan; Debug; Ask 
```

* Ответ

```text
Ситуация с отсутствием пункта «Add custom Mode» в Cursor 2.3 связана с тем, что 
разработчики официально удалили (deprecated) функцию «Custom Modes» начиная с версии 2.1.

Вот основные причины и то, как теперь предлагается работать:

1. Почему это произошло?

Разработчики Cursor решили, что концепция «Кастомных режимов» была избыточной и сложной в поддержке. 
Вместо создания отдельных «режимов» они перешли к системе Custom Commands (Кастомные команды) и 
расширению возможностей Agent.

Теперь логика такая:

    Agent — это универсальный «мозг», который сам решает, какие инструменты использовать.

    Plan — для обсуждения архитектуры без написания кода.

    Debug — для исправления ошибок.

    Ask — для простых вопросов.

2. Как теперь делать «свои режимы»?

Если вы хотите воссоздать функционал Memory Bank или свои специфические инструкции, 
используйте один из двух путей:


Способ А: Custom Commands (рекомендуемый)

Вместо режима в выпадающем списке теперь используются команды через слэш (/).

    Создайте файл в папке .cursor/commands/my-command.md (или настройте в Settings).

    В этом файле пропишите системный промт и инструкции.

    В чате просто пишите /название_команды и ваш запрос. ИИ применит все инструкции этой команды.


Способ Б: Правила для конкретных папок (.cursorrules)

Если вам нужно, чтобы ИИ вел себя по-особенному в конкретном проекте (например, всегда обновлял Memory Bank):

    Создайте файл .cursorrules в корне проекта.

    Пропишите там: "Ты всегда работаешь в режиме [Название]. Твои задачи: 1. Проверить context.md... 2. Сделать..."

    Теперь любой запрос в обычном чате будет проходить через фильтр этих правил.


Способ В: Настройки Features

Проверьте на всякий случай: Settings (Ctrl + Shift + J) -> Features -> Chat. 
В некоторых переходных версиях там был тумблер «Enable Custom Modes (Beta)». 
Если его нет — значит, в вашей сборке функция окончательно заменена на Commands.

Что делать с Memory Bank?

Если вы скачали репозиторий vanzan01/cursor-memory-bank, он уже адаптирован под новую реальность. 
Вместо выбора режима в меню, вы просто используете команды:

    /van — для инициализации.

    /plan — для планирования.

Хотите, я помогу составить текст для вашей первой кастомной команды (.md файл), 
которая будет имитировать нужный вам режим?
```
